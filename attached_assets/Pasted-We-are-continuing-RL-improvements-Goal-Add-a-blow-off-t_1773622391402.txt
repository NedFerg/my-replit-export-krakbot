    0 = normal

    1 = elevated (early warning)

    2 = blow‑off (de‑risk aggressively)

    use existing regime features (no new data sources)

    tighten dynamic caps when top_risk rises

    adjust reward shaping to discourage adding risk into a blow‑off

    generate feedback messages like:

        “Top‑risk rising: early blow‑off conditions detected.”

        “Blow‑off top conditions detected — de‑risking recommended.”

All RL core logic stays unchanged.

    score += 1.0

    score += 1.0

# 2. Volatility rising while prices are strong → instability

    score += 1.0
if vol > 0.025:
    score += 1.0

# 3. Liquidity thinning → exhaustion

    score += 1.0

    self.top_risk = 0   # normal

if not hasattr(self, "prev_top_risk"):
