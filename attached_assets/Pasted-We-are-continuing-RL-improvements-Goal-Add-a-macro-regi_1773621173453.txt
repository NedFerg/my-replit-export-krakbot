4. Make reward regime‑aware

In simulation.py, where you compute the RL reward (portfolio return minus costs/penalties), modify it to depend on macro_regime, altseason_index, vol_regime, and turnover.

Assume you already have:

    step_return (portfolio return)

    txn_cost (sum across assets)

    turnover (sum of |Δexposure| across assets)

    state.altseason_index, state.vol_regime, state.liq_regime, state.macro_regime

Add something like:
python

macro = self.macro_regime
alt = getattr(state, "altseason_index", 0.0)
vol = getattr(state, "vol_regime", 0.0)

base_reward = step_return - txn_cost

# Regime-aware turnover penalty
if macro > 0:
    turnover_penalty = 0.001 * turnover  # looser in bull
elif macro < 0:
    turnover_penalty = 0.005 * turnover  # tighter in bear
else:
    turnover_penalty = 0.003 * turnover

# Extra upside weight in bull + strong downside penalty in bear
if macro > 0:
    # reward upside more when altseason is strong
    reward = base_reward * (1.0 + 0.5 * alt) - turnover_penalty
elif macro < 0:
    # penalize negative returns more when vol is high
    downside = min(step_return, 0.0)
    reward = base_reward - 2.0 * abs(downside) * (1.0 + 5.0 * vol) - turnover_penalty
else:
    reward = base_reward - turnover_penalty

Use your judgment to tune the coefficients; the key is:

    Bull: encourage upside capture, lighter turnover penalty

    Bear: heavily penalize drawdowns and churn

    Neutral: middle ground

5. Add a macro feedback module

In simulation.py, after computing regime features (btc_dominance, altseason_index, vol_regime, liq_regime) and knowing self.macro_regime, add a simple diagnostic block that logs human‑readable feedback.

At the top of simulation.py, add a list to store messages:
python

self.macro_messages = []

Then each step, after regime computation:
python

macro = self.macro_regime
alt = self.altseason_index
vol = self.vol_regime
liq = self.liq_regime
btc_dom = self.btc_dominance

message = None

# Example conflict conditions
if macro > 0:
    # You say bull, but signals look stressed
    if vol > VOL_HIGH_THRESHOLD and alt < ALT_LOW_THRESHOLD and btc_dom_rising_condition:
        message = "Signals conflict with your macro call: bull vs stress (high vol, weak alts, rising BTC dominance)."
elif macro < 0:
    # You say bear, but signals look risk-on
    if vol < VOL_LOW_THRESHOLD and alt > ALT_HIGH_THRESHOLD and btc_dom_falling_condition:
        message = "Signals conflict with your macro call: bear vs risk-on (low vol, strong alts, falling BTC dominance)."

if message:
    self.macro_messages.append(message)
    print(message)

Define simple thresholds near the top of the file:
python

VOL_HIGH_THRESHOLD = 0.02   # example, tune based on your vol scale
VOL_LOW_THRESHOLD  = 0.005
ALT_HIGH_THRESHOLD = 0.6
ALT_LOW_THRESHOLD  = 0.2

And simple BTC dominance conditions, e.g.:
python

btc_dom_rising_condition = self.btc_dominance > 0.5
btc_dom_falling_condition = self.btc_dominance < 0.4

The exact numbers can be tuned; the important part is:

    When your macro flag and the signals disagree, a clear message is printed and stored.
