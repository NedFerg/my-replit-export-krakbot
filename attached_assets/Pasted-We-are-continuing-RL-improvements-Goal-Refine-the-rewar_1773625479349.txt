3. Add drawdown penalties to reward

After your existing reward logic (regime, blow‑off, panic, rotation, cycle):

# --- Drawdown penalties ---
# 1. Depth penalty (encourage shallow drawdowns)
reward -= 2.0 * self.drawdown

# 2. Acceleration penalty (punish rapidly worsening drawdowns)
dd_delta = self.drawdown - self.prev_drawdown
if dd_delta > 0:
    reward -= 5.0 * dd_delta

This teaches the agent:

    avoid deep drawdowns

    avoid accelerating losses

    cut risk early

4. Add turnover efficiency scoring

After drawdown penalties:
python

# --- Turnover efficiency ---
# If turnover is high but return is low → penalize
if turnover > 0.02 and abs(step_return) < 0.002:
    reward -= 0.5 * turnover

# If turnover is low and return is positive → reward stability
if turnover < 0.01 and step_return > 0:
    reward += 0.1 * step_return

This encourages:

    fewer pointless trades

    more conviction

    smoother exposure patterns

5. Add stability bonus for consistent profitable states

After turnover efficiency:

# --- Stability bonus ---
# Reward being near new highs
if self.equity >= 0.995 * self.peak_equity:

This is tiny but directional — it nudges the agent toward:

    maintaining gains

    avoiding unnecessary churn

    respecting momentum
