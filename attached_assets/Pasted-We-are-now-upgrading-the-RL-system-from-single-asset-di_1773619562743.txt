3. Adapt the critic to actor–critic

You currently have a C51 dueling network with discrete actions.

Refactor it into a state–value critic:

    Modify the C51 network to output V(s) instead of Q(s, a):

        Remove the action dimension

        Output a distribution over returns for each state only

    Update the RL training loop:

        Use the C51 critic as a value estimator:

            V(s) from the distribution (e.g., expectation over atoms)

        Use the actor to generate actions

        Use TD targets with C51 for value loss

    Policy loss:

        For each sampled transition:

            Compute V(s) and V(s')

            Compute advantage A ≈ r + γ V(s') − V(s)

            Policy loss: L_actor = -E[ A * log_prob(action | s) ]

        Backprop through self.actor only

    Keep all existing C51 machinery (atoms, support, projection) but treat it as a V(s) critic, not Q(s, a).
