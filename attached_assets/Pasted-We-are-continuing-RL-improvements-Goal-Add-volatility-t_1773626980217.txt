Goal:  
Add volatility targeting, so the system automatically scales its exposures to maintain a stable risk level.

This module adds:

    rolling realized volatility

    a target volatility (e.g., 12% annualized)

    a scaling factor applied to all exposures

    reward shaping to encourage stable volatility

    feedback messages when volatility deviates too far from target

All RL core logic stays unchanged.

self.vol_scaler = 1.0
self.prev_realized_vol = 0.0

2. Compute rolling realized volatility

After computing step_return each step:
python

# --- Rolling realized volatility (30-step window) ---
window = self.equity_history[-30:]
if len(window) >= 2:
    rets = [(window[i] - window[i-1]) / max(window[i-1], 1e-6) for i in range(1, len(window))]
    mean_r = sum(rets) / len(rets)
    var_r = sum((r - mean_r)**2 for r in rets) / len(rets)
    self.prev_realized_vol = self.realized_vol
    self.realized_vol = (var_r ** 0.5) * (252 ** 0.5)  # annualized
else:
    self.realized_vol = 0.0

Attach to state again:
python

setattr(state, "realized_vol", self.realized_vol)

This keeps scaling between 0.25× and 2× — enough flexibility without destabilizing training.

5. Add volatility‑aware reward shaping

After your existing reward logic (drawdown, turnover, stability):
python

# --- Volatility targeting reward ---
vol_error = abs(self.realized_vol - self.target_vol)

# Penalize large deviations from target vol
reward -= 0.5 * vol_error

# Bonus for staying close to target
if vol_error < 0.02:
    reward += 0.002

This teaches the agent to:

    avoid volatility spikes

    avoid volatility collapse

    maintain consistent risk
